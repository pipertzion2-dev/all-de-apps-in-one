    FAQ

    What does the Hardware Documentation Exporter do?

    It runs one isolated step of Svivva's real pipeline: you describe your concept (product, music hardware, or build idea), and the tool returns a production-style preview — schematic, BOM table, flow graph, idea report, or setup map — plus a readiness indicator, so you can see how your docs map to the full Svivva workflow.

    Is it free and do I need to sign up?

    Yes. This exporter is completely free and requires no login. You can generate previews as often as you like. To run the full workflow with versioning and cost tracking, use the main Svivva platform.

    How does this map to the full Svivva workflow?

    This tool exposes one step (e.g. Concept → Schematic, Intent → Clip, Idea → Report). In Svivva, the same step sits inside the full flow: hardware pipeline, BOM generation, schematic layout, cost tracking, and production-ready exports — with versioning and collaboration.

    What types of hardware documentation can I export?

    The tool supports six modes: Hardware (schematic + BOM preview), Build (3D placeholder + BOM), Play (audio and setup map), Idea (idea report and doc summary), Team (collaboration setup map), and Marketplace (flow graph). Each mode generates documentation suited to that stage of hardware development in Svivva.

    Can I use this for BOM generation?

    Yes. In Hardware and Build modes, the exporter generates a Bill of Materials table with estimated unit costs, quantities, and a total with contingency. For full BOM management with supplier data, version history, and cost tracking, use Svivva's complete hardware pipeline.

    What is the readiness score?

    The readiness indicator shows how production-ready your documentation is based on how much detail you provide. Adding constraints like budget, materials, and region increases the score. In Svivva, readiness tracking is integrated across every stage of your hardware pipeline.
