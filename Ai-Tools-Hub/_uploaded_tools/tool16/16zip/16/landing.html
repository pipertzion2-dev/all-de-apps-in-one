FAQ

Common Questions About JSON Schema Validation

What is a JSON Schema validator?

A JSON Schema validator checks whether a JSON document conforms to a schema definition — verifying required fields are present, types are correct, and values satisfy constraints like enums and ranges. The free linter on this site goes a step further by scoring how tightly your schema is written and recommending specific improvements.

What is the difference between a JSON Schema linter and a validator?

A validator checks data against a schema. A linter checks the schema itself — flagging missing required fields, overly permissive types, and deep nesting that will cause problems at runtime. Use the JSON Schema Linter to improve your schema definition, then use Svivva to validate live traffic against it.

Does Svivva support OpenAPI 3.x and Swagger?

Yes. Svivva supports JSON Schema Draft 4–7, OpenAPI 3.x, and Swagger 2.0. You can import a spec directly from a URL, paste it manually, or connect a GitHub repository for continuous schema tracking.

How does schema tightness affect LLM and AI tool calling?

When using OpenAI function calling, Anthropic tool use, or any LLM agent framework, the model reads your schema to decide which fields to include and what values to generate. Loose schemas — with too many optional fields or vague string types — directly cause hallucinated values, missing required data, and function call failures. Tighter schemas produce more reliable AI agent behavior. See the free tightness scorer to benchmark yours.
