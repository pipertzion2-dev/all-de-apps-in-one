1. Authentication Tests

  2. Try to access /dashboard/projects/new directly
Expected: User is redirected to login page or shown login prompt.

2. API Creation Flow

TC-2.1: Navigate to Create New API

  1. Log in to the application
  2. Click "Create New API" or navigate to /dashboard/projects/new
Expected: API Creator panel appears with step indicator showing DEFINE - CONFIGURE - FINISH

TC-2.2: DEFINE Phase - Enter API Details

  1. Enter a prompt describing the API (e.g., "Analyze sentiment of customer reviews")
  2. Enter expected output format (e.g., "{ sentiment: positive/negative/neutral, score: 0-100 }")
  3. Click the "Tune" button
  4. Adjust sliders: Creativity, Detail Level, Schema Strictness
  5. Toggle: Confidence Scores, Metadata options
  6. Click "Continue" or "Next"
Expected: All inputs accept text. Sliders show real-time feedback. Toggles work. Advances to CONFIGURE phase.

TC-2.3: CONFIGURE Phase - Answer Questions

  1. Review each configuration question presented
  2. Provide answers for each question
  3. Navigate through all questions
  4. Click "Continue" to proceed
Expected: Questions are relevant to the API prompt. Progress indicator updates. Advances to FINISH phase.

TC-2.4: FINISH Phase - Set Identity

  1. View AI-suggested names for the API
  2. Select or enter a custom name
  3. Choose an icon from available options
  4. Select a color palette
  5. Observe live preview card updating
  6. Click "Create API" button
Expected: Preview card updates in real-time. API is created successfully. Redirects to project detail page.

3. API Tuner Controls

TC-3.1: Creativity Slider

  1. Open the Tune panel
  2. Move Creativity slider from 0% to 100%
  3. Observe label text changing
Expected: Label changes from "Conservative & predictable" to "Creative & experimental"

TC-3.2: Toggle Switches

  1. Click "Include Confidence Scores" toggle
  2. Click "Include Metadata" toggle
  3. Toggle each on and off
Expected: Toggles switch states visually. Settings are retained.

4. Visual & UI Tests

TC-4.1: Dark Mode Toggle

  1. Locate theme toggle (sun/moon icon)
  2. Click to switch between light and dark mode
  3. Navigate through different pages
Expected: All pages render correctly in both modes. Text is readable. No broken colors.

TC-4.2: Responsive Layout (Desktop)

  1. View app at 1920x1080 resolution
  2. Resize browser to 1280x720
  3. Check all major pages
Expected: Layout adjusts appropriately. No horizontal scrolling. All elements visible.

TC-4.3: Step Indicator Navigation

  1. Observe step indicator: DEFINE - CONFIGURE - FINISH
  2. Complete each phase
  3. Verify current step is highlighted
Expected: Current step is visually distinct. Completed steps show checkmark or different style.

5. Error Handling

TC-5.1: Empty Form Submission

  1. Try to proceed without entering a prompt
  2. Try to proceed without entering expected output
Expected: Validation messages appear. Form does not submit until required fields are filled.

TC-5.2: Network Error Handling

  1. Disconnect from internet (or simulate offline)
  2. Try to create an API
  3. Reconnect and retry
Expected: Error message displays gracefully. App recovers when connection restored.
